# 文献内容
本研究提出了 **MAGVIT-v2**，一种新型视频分词器（Tokenizer），旨在通过改进离散表示学习，使大语言模型（LLMs）在图像和视频生成质量上超越扩散模型 。
### 1. 必要的前置背景与核心挑战
- **视觉生成中的 LM vs. Diffusion**：虽然 LLMs 是文本生成的统治者，但在视觉领域长期逊色于扩散模型（如 ImageNet 上的 FID 差距曾高达 48%）。
- **离散表示的局限性**：LLM 处理视觉数据需先通过分词器将像素映射为离散 Token 。以往的 VQ-VAE 类模型在词表扩充时，往往会导致生成质量下降（如图 1 所示的橙色曲线），这限制了 LM 建模复杂视觉分布的能力 。
- **时空一致性**：现有分词器（如 MAGVIT-v1）在处理单图和长视频时存在困难，且视频生成常伴随闪烁现象 。
### 2. 核心解决的问题
- **词表扩展难题**：如何让分词器在词表增加（如增加到 262,144）的同时，依然保持 LM 的生成效率与质量
- **统一模态表示**：构建一个能够同时兼容图像和视频，且在时空维度上具备因果性的统一分词器
### 3. 关键数学推导：无查找量化 (Lookup-Free Quantization, LFQ)
为了克服传统矢量量化（VQ）中词表嵌入维度的限制，MAGVIT-v2 将嵌入维度降为 0，通过对编码器输出进行二值化处理来映射 Token 。
**变量定义：**
- $K$：词表大小 。
- $z \in \mathbb{R}^{\log_2 K}$：编码器输出的潜在特征向量 。
- $q(z)$：量化后的特征向量 。
- $C_i = \{-1, 1\}$：第 $i$ 维的二值码本空间 。
**量化过程：** 利用 $sign$ 函数进行逐维度量化，每个维度相互独立 ：
$$q(z_i) = sign(z_i) = \begin{cases} 1 & z_i > 0 \\ -1 & z_i \le 0 \end{cases}$$
**Token 索引计算：** 通过位偏移将二值向量转换为整数索引 ：
$$Index(z) = \sum_{i=1}^{\log_2 K} 2^{i-1} \mathbb{I}\{z_i > 0\}$$
**训练目标（熵损失）：** 为了保证词表利用率，引入熵惩罚项 ：
$$\mathcal{L}_{entropy} = \mathbb{E}[H(q(z))] - H[\mathbb{E}(q(z))]$$
- **左项 $\mathbb{E}[H(q(z))]$**：最小化单个样本的熵，使量化结果尽可能确定 。
- **右项 $H[\mathbb{E}(q(z))]$**：最大化平均输出的熵，鼓励均匀使用所有 Token 。

### 5. 实验结果与启示
#### 5.1 实验数据
- **图像生成 (ImageNet 512x512)**：MAGVIT-v2 (MLM) 达到 **FID 1.91**，显著优于 DiT-XL/2 (FID 3.04) 和 VDM++ (FID 2.65) 。
- **视频生成 (Kinetics-600)**：FVD 指标降至 **5.2**，远超其前作 MAGVIT-v1 (FVD 9.9) 。
- **视频压缩**：人类评测显示其压缩质量优于 HEVC，并可与下一代标准 VVC (H.266) 媲美 。
#### 5.2 核心启示
- **Tokenizer 是关键**：LM 与扩散模型的差距并非源于 Transformer 架构本身，而是由于分词器未能提供高效、高容量的视觉表征 。
- **大词表的价值**：通过 LFQ 实现的大词表（如 $2^{18}$）能够有效捕捉视觉细节，且不会像传统 VQ 词表那样使训练变得不稳定 。
- **统一建模的可能性**：使用同一套词表处理图像、视频生成、压缩及识别任务是可行的，这为真正的多模态 LLM 奠定了基础 。
# 关键启示
- 一个架构的重建误差和生成误差不一定是相关的，甚至可能是负相关的，因此在设计分词器时，不能仅仅关注重建误差，还需要关注生成误差。此处的LFQ方法改善了原来的VQ方法在生成误差上的表现，从而使得LM在视觉生成任务上能够超越扩散模型
	- 此处做了省去look-up的二进制编码式的encoding，有效解决了这一问题！
- 视频生成的**因果关系**意味着时间维度是单向的，和其他的双向维度不一样，因此在设计模型时需要特别考虑时间维度的处理方式，如使用时空因果3D CNN架构来确保视频生成的连续性和兼容性。
	- 时间维度的压缩不宜过早
	- 可参考的架构：[[Recurrent Convolutions for Causal 3D CNNs.pdf|Casual 3D CNN]]
- *和预想的类似，tokenize的思路更适合处理时间上的因果关联，和气象模型的降水时序预测思路有相似之处。或许可以考虑这个方向，也可以参考后续有没有结合diffusion和tokenize的模型，来看看是否有更适合气象场建模的架构设计。*
