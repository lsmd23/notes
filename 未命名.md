**1. 总体成果**

你当前完成的是：在原 NowcastDiT 的 VAE latent 框架上，把原本 full-sequence DiT 改造成一个近似 Diffusion Forcing 思路的版本。

原项目更接近：

```text
历史帧 + 未来帧整体序列
-> 同一个 diffusion timestep
-> DiT 全序列 attention
-> 一次性学习完整未来序列的去噪
```

你现在的版本变成：

```text
VAE latent 序列 z_1:T
-> 每个 frame 独立采样 noise level k_t
-> history latent 保持 clean
-> future latent 独立加噪
-> DiT 接收 (B,T) timestep
-> block-wise causal attention
-> 只在未来帧上计算 noise prediction loss
```

这已经抓住了 DF 的关键思想：**不同时间 token 可以处在不同噪声等级，模型学习在不确定时间序列上做逐 token 的去噪建模**。

但也要实事求是地说：这还不是原文那种严格 RNN hidden-state causal DF。当前是 **DF-like DiT adaptation**，也就是把 DF 的 per-frame noise level 和 causal mask 迁移到 DiT 结构里。

**2. 如何把 Diffusion Forcing 迁移到 DiT**

核心实现集中在：

```text
model/df_dit/denoiser.py
model/df_dit/video_dit.py
```

在 `DFLikeDenoiser` 里，你实现了：

```python
t_idx = torch.randint(0, num_timesteps, (B, T))
```

这对应 DF 里的：

```text
k_t ~ uniform[K]
```

然后对每一帧 latent 独立加噪：

```text
x_t^k = sqrt(alpha_k) x_t + sqrt(1-alpha_k) eps_t
```

同时保留历史条件：

```python
t_idx[:, :cond_len] = 0
x_t[:, :cond_len] = x_1[:, :cond_len]
```

loss 只算未来：

```python
MSE(pred_noise[:, cond_len:], noise[:, cond_len:])
```

这对应你最初关注的公式：

```text
sum_t || eps_t - eps_theta(z_{t-1}, x_t^{k_t}, k_t) ||^2
```

只不过当前的 `z_{t-1}` 不是 RNN hidden state，而是通过 **causal/block-causal self-attention 对历史 token 的可见性约束** 来近似实现。

**3. 如何处理现有 DiT**

你没有直接破坏原 `model/dit`，而是在：

```text
model/df_dit/
```

下隔离了一份 DF DiT。这个选择是正确的，因为它保留了原 baseline，同时方便后续对比。

DiT 的关键改造包括：

```text
(B,) timestep -> 支持 (B,T) timestep
full attention -> 支持 block causal mask
原始 full-sequence noise -> per-frame independent noise
```

在 `video_dit.py` 中，`_prepare_timestep_conditioning` 做了两层处理：

1. 每帧 timestep embedding 加到对应 frame 的 patch tokens 上；
2. 对 `(B,T)` timestep 求 mean，得到一个 global timestep summary，用于 AdaLN。

也就是说：

```text
frame-level noise condition -> token embedding
global sequence noise condition -> AdaLN modulation
```

这个设计很适合从原 DiT 平滑迁移，因为原 DiT block 本来就是 AdaLN-conditioned transformer block，不需要完全重写架构。

条件输入方面，你继续使用 `DiTConditioner`，把 history/future condition latent 编成和 patch token 同分辨率的条件特征；future 部分通过 `_mask_condition` 置零或替换为 unconditional token，避免直接泄漏未来真值。

**4. 后端 Attention 的处理**

你现在不只是加了一个 dense mask，而是做了 attention backend 抽象：

```text
model/df_dit/attention.py
```

当前支持：

```text
eager
sdpa
flash
flex
block_sparse
axial
```

最关键的是 `block_sparse`。因为普通 SDPA 对 arbitrary mask 往往会退回 math backend，显存和速度都不理想。你现在的 block sparse 思路是：

```text
按时间 block 切分 query
每个 query block 只 attend 自己和过去 block
对每个 block 调用 SDPA
```

这对应：

```text
causal_block_size: 5
```

也就是 25 帧被分成 5 个时间 block：

```text
0-4, 5-9, 10-14, 15-19, 20-24
```

后面的 block 不能看未来 block。这比严格逐帧 causal 更宽松，但也更适合 DiT 并行训练，是一个合理工程折中。

你还保留了 FlexAttention 路径：

```python
from torch.nn.attention.flex_attention import create_block_mask, flex_attention
```

如果未来 PyTorch 版本支持更成熟，可以把 block mask 下放给 FlexAttention；当前 `block_sparse` 是一个可运行、可控、容易 debug 的方案。

**9. 现有主要问题**

第一，当前还不是严格 Diffusion Forcing 原文结构。原文强调因果 hidden state `z_{t-1}`，你现在是用 block-causal attention 近似历史总结。这个迁移是合理的，但理论上仍和 RNN-style DF 有距离。

第二，长期预测衰减明显。可视化中前 5-10 帧不错，后 10 帧变弱；指标里高阈值后半程基本归零。这说明模型学到了短期外推，但长期不确定性和强回波维持还不足。

第三，高阈值强回波仍然偏弱。虽然 `CSI@181`、`CSI@219` 已经非零，但主要集中在前几帧，后期几乎消失。

第四，评估样本数需要澄清。你当前给出的日志是 `Total 192 samples`，如果要正式汇报，最好补一个完整 validation eval，或者明确说这是 sampled validation / quick eval。

第五，attention 后端还有工程优化空间。`block_sparse` 已经比 dense mask 更可用，但本质还是 Python-level block loop + SDPA，未来如果能接入成熟 FlexAttention/Triton block sparse，会更干净也更快。

第六，VAE latent cache 固定了 tokenizer，这是好事，但也意味着所有误差最终受 VAE 表达能力限制。强回波细节如果在 VAE latent 中已经损失，DiT 很难完全恢复。
