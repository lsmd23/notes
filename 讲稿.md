# 第 5 页：项目背景：研究问题

**建议时长：70 秒**

我的项目背景是雷达回波预测的气象模型。在这一方面，我了解到课题组内已经有相当的进展：在NowcastNet 上，结合气象任务先验，模型已经具有了极端降水预测和真实预报的能力。

与此同时，近年来，DiT、video diffusion 这类生成式时空模型发展很快，它们在一般的图像上已经具有了强的全局建模能力和不确定性建模能力，而且能够处理复杂的时空依赖。但如果直接迁移到雷达预测，又会缺少气象约束，可能带来时间因果性不足、虚警增多、高回波低估等问题。

我的小项目主要围绕一个问题：能否利用现有视频生成模型对复杂空间依赖和时间依赖的建模能力，将其迁移到雷达预测任务中。因为雷达预测本质上不是普通的一维时间序列预测，而是一个高维时空场的未来生成问题。每一帧雷达图像都包含复杂的降水区域、边界和强度结构；连续多帧之间又存在移动、增强、消散、分裂和合并等动态过程。因此模型既要理解空间结构，也要保持时间一致性。

从现有方法看，Auto-regressive 的生成适合逐步外推建模时间因果性，但容易在时间维度上误差累积；Full-sequence diffusion 有较强的全局生成能力，但如果直接用于气象预测，又可能难以处理更强的时间因果依赖。

所以我的理解是，雷达预测需要在两类能力之间做平衡，这也是后面尝试 Diffusion Forcing 和 研究 causal / noncausal attention 对比和细节的出发点。

# 第 6 页：项目背景：相关方法基础

**建议时长：60 秒**

这个项目基于三个主要方法进行：

第一是 Diffusion Forcing。它的核心思想是把 next-token prediction 和 full-sequence diffusion 统一到 per-token noise 的框架中。也就是说，序列中不同 token 可以处在不同噪声等级，模型学习在这种不均匀噪声状态下进行去噪和生成。

第二是 History Guidance。它关注的是利用已经生成或已有的历史帧来引导后续视频传播，从而缓解长时程扩散生成中的误差积累。这一思路借鉴了Conditional-Free Guidance的思路，将历史信息作为条件引导输入，从而控制AG生成误差。

第三是 MM-DiT 或 DiT 类架构。它用 Transformer denoiser 替代传统 U-Net，通过 attention 机制建模全局 token 关系，并利用条件调制增强生成质量。这一项目就是在这个基础上，尝试把 Diffusion Forcing 的 per-frame noise、history guidance 和 attention 机制分析结合起来。

---

# 第 7 页：模型架构

**建议时长：70 秒**

这一页是模型整体结构。输入是历史和未来组成的 radar sequence。首先通过一个已经充分训练的 VAE encoder，把原始雷达帧压缩到 latent sequence 中。这样做的好处是减少像素空间直接建模的计算压力，也让核心 DiT 模块可以在 latent 空间中学习时空生成规律。

中间核心模块是 Diffusion Forcing DiT。它接收三类信息：第一是 latent sequence；第二是每一帧独立采样的 noise level，也就是 per-frame timestep；第三是 history guidance level 和 attention mask，用来控制历史帧与未来帧之间的信息流。

右侧是 DiT 内部结构。Embedding 和 global AdaLN summary 负责把噪声等级和条件信息注入到 Transformer block 中。每个 block 主要包括 LayerNorm、Multihead Attention 和 MLP，并重复 L 层。模型输出的是 predicted noise，也就是学习在不同噪声等级下对 latent sequence 进行去噪。

训练时使用 noise prediction loss；推理时采用 DDIM 采样方法，结合不同的 inference schedule 的效果，完成对未来雷达帧的生成。

# 第 8 页：模型训练与初步结果

**建议时长：70 秒**

这一页是训练设置和初步结果。我在 SEVIR 数据集上训练了两个主要模型：一个是 causal HG fine-tuning，一个是 noncausal HG fine-tuning。

causal 模型使用 block causal attention，block size 设为 5，相当于人为加入了块级时间因果约束，主要用来验证因果约束在 attention 机制和预测结果中的影响。noncausal 模型则使用 full attention，让未来 noisy token 之间也可以充分通信，是DFoT模型原有的方法，可以验证全局 noisy future context 的生成能力。

训练上，我固定已经充分训练的 VAE，不再更新 encoder 和 decoder，只缓存 latent，然后训练核心 DF-DiT 模块。模型规模大约是 45.29M 参数。当前 causal 模型训练到 base training 200 epoch 加 history guidance fine-tuning 30 epoch。

从指标看，noncausal 在 CSI、HSS、MSE、MAE 和 POD 上整体更优，比如 CSI 从 causal 的 0.1170 提升到 0.1673，MSE 也明显下降。但 noncausal 的 FAR 更高，说明它召回更多强回波和降水区域的同时，也更容易产生虚警。这个现象为后面的 attention 分析提供了动机。

# 第 9 页：结果分析——Inference Scheduling

**建议时长：55 秒**

下来深入分析训练结果和可能的原因：

这一页分析 inference scheduling 的影响。因为 Diffusion Forcing 的特点是每一帧可以有不同噪声等级，所以推理时如何安排不同时间帧的去噪进度，会直接影响最终生成质量。

我比较了 synchronized、random matrix 和不同 shift 的 causal pyramid schedule。结果显示，调度矩阵对结果有明显影响。其中 causal pyramid shift=0.5 的综合结果最好，CSI 达到 0.1747，HSS 达到 0.2304，MSE 也降到 437.47。

我的理解是，pyramid schedule 更符合序列预测中的“逐步向未来传播”的生成范式。它不是让所有帧完全同步去噪，而是让不同未来帧处在不同生成阶段，从而更好地利用已经较可靠的近未来信息去引导更远未来。这和 DFO T / Diffusion Forcing 中强调的 flexible horizon 和 history-conditioned generation 是一致的。

# 第 10 页：结果分析——Attention Visualize

**建议时长：55 秒**

这一页开始分析 attention。这里我主要比较 causal 和 noncausal 模型在 history 依赖上的差异。

从表格可以看到，causal 模型在各个 block 上的 history mass 明显更高，例如 block 1 到 block 3 基本都在 0.60 左右。这说明 causal attention 在结构上确实迫使模型更多依赖历史信息，形成一种 history-anchored 的去噪方式。

相比之下，noncausal 模型的 history mass 只有 0.03 到 0.07 左右，说明它并不是主要依赖历史帧，而是更分散地使用整个序列上下文，尤其是 noisy future token 之间的信息。

这个结果和前面的指标是对应的：causal 更保守、更依赖历史，可能更稳定；noncausal 更开放，更善于利用未来 noisy context，因此召回能力更强，但也带来了更高的虚警风险。

# 第 11 页：结果分析——Noise Level Attention

**建议时长：65 秒**

这一页进一步分析 attention 和噪声等级之间的关系。Diffusion Forcing 的关键是每个时间 token 都可以有独立噪声等级，所以我把 key token 按 clean、low、mid、high noise 分组，观察模型在不同 sampling step 上到底更关注哪类 token。

从表格可以看到，causal 模型的 clean key mass 明显高于 noncausal。比如 step 0 时，causal 有 0.795 的 attention mass 落在 clean key 上，而 noncausal 是 0.576；到 step 49，causal 仍然有 0.468，而 noncausal 只有 0.291。这说明 causal 模型更依赖 clean history 和已经较可靠的上下文。

相反，noncausal 的 expected key noise 在 early 和 middle step 更高，比如 step 10 是 0.429，而 causal 只有 0.235。这说明 noncausal 更愿意利用带噪的 future token，让未来帧之间互相协商结构。

我的结论是，per-frame noise 结构确实在 attention 内部产生了可观测的信息流差异：causal 更像 history-anchored denoising，noncausal 更像 future-context collaborative denoising。这也解释了为什么 noncausal 的 CSI 和 POD 更高，但 FAR 也更高。

# 第 12 页：结果分析——Attention Patterns

**建议时长：60 秒**

这一页是 head-level 的 attention pattern 分析。我在阅读 video model 相关工作时注意到，很多视频模型会出现不同功能的 attention heads，比如 temporal heads 和 spatial heads。所以我希望看看，在这个雷达 DF-DiT 模型中是否也出现了类似分化。

这里每个点代表一个 attention head，我用 temporal focus、spatial focus、self mass 和 history block mass 来描述它的功能倾向。结果显示，模型确实有 head specialization，但主要体现在 temporal 和 history 维度上。

比如 causal 模型后几层的 history block mass 大约在 0.60 左右，说明它形成了比较强的历史锚定；而 noncausal 的 history block mass 只有 0.03 到 0.07，更偏向全序列开放交互。

不过，一个比较重要的发现是：模型没有表现出明显的 spatial-like head。我的理解可能有几方面原因：第一，当前是在固定 VAE latent 上建模，空间分辨率已经被压缩；第二，当前 attention 设计没有显式的空间指向机制；第三，雷达预测这个阶段的主要难点可能更集中在时间演化和回波移动上。

所以这一页的结论是：当前模型已经学到了一定 head 分化，但分化主要是 temporal/history specialization，而不是明显的 spatial/temporal 二分。这也提示后续可以继续优化 attention 结构，让模型更好地建模局部空间运动。

# 第 13 页：项目展望

**建议时长：60 秒**

最后是项目展望。我把后续方向分成三个层次。

第一是更严谨的评估与可复现实验。当前实验 case 数量还比较有限，后续需要扩展到更稳定的验证集，并系统消融 schedule shift、history guidance scale、causal block size 和 attention backend 等因素。

第二是继续优化推理机制和可解释分析。尤其是 history guidance 中 conditional 和 unconditional branch 的行为差异，以及训练时 per-frame noise sampling 和推理 schedule 之间是否一致。这些都会影响模型是否能稳定地长程外推。

第三是面向雷达任务本身的模型改进。比如高回波区域目前仍然是难点，后续可以分析高回波低估或虚警的具体来源，探索加权损失、高回波 mask、雷达先验融合等方法。同时，attention 结构也可以继续优化，使它不仅关注时间维度，也更好地捕捉局部空间运动。

更长期看，我希望从“视频生成模型迁移到气象数据”进一步走向“面向气象任务定制生成模型”，包括更适合气象序列的 latent tokenizer、spatiotemporal attention，以及多变量、多模态气象大模型。
