- 上层工具发送的SQL请求，都交由查询处理器（Query Processor）来处理，查询处理器的职责是解析、优化和执行SQL请求
- 查询执行引擎（Query Execution Engine）是查询处理器的一个组件，负责执行查询计划并返回结果，同时尽可能地优化查询性能
# 概述
- 基础：[[L1 关系模型#关系代数基础|关系模型——关系代数基础]]
- 查询处理的步骤：
	1. 语法分析与翻译（Parser and Translator）：将SQL查询转换为关系代数表达式
		- Parser会首先进行语法检查，验证语法正确性
		- 之后会进行语义验证，确保查询中的表、列等元素存在且正确使用
		- 最后会将SQL翻译为扩展关系代数表达式，或者对应的查询树
		- 例：SQL语句：`SELECT name FROM instructor WHERE salary < 75000;`将被翻译成关系代数：$\Pi_\text{name}(\sigma_{\text{salary} < 75000}(\text{instructor}))$，同时生成对应的查询树
	2. 查询优化（Optimizer）：在所有查询方案中，选出代价最低的一个
		- 逻辑优化（代数优化）：对关系代数表达式做等价变换，使其更加高效
			- 例：关系代数表达式$\Pi_\text{name}(\sigma_{\text{salary} < 75000}(\text{instructor}))$和$\sigma_{\text{salary} < 75000}(\Pi_\text{name}(\text{instructor}))$是等价的，但前者通常更高效，因为它先过滤掉不满足条件的元组，再进行投影
		- 物理优化：选择具体的算法和数据结构来执行查询计划
			- 例：选择操作有全表扫描和索引扫描两种算法，优化器会根据表大小和索引清空选择更高效的算法
	3. 查询执行（Evaluation Engine）：
