# 物理存储介质基础
- 计算机的存储结构：参见[[L4.1 存储器体系结构#存储器结构抽象|计算机组成原理——存储器结构抽象]]与[[L4.2 缓存友好的代码#缓存组织和操作|计算机组成原理——缓存组织和操作]]
- 存储介质特性与分类：
	- 存储评价指标：
		- 访问速度：数据读取/写入的耗时
		- 单位成本：每字节存储的成本
		- 可靠性：抗断电，抗系统崩溃和物理故障的能力
	- 介质分类：
		- 易失性存储（volatile storage）：如RAM，断电后数据丢失，包括Cache和主存
		- 非易失性存储（non-volatile storage）：如磁盘，断电后数据仍然存在，包括HDD和SSD
	- 各存储介质特性：
	- |介质类型|核心特性|访问速度|适用场景|
		|---|---|---|---|
		|**Cache**|最快、成本最高、硬件管理|纳秒级|CPU 高频数据缓存（L1/L2/L3）|
		|**Main Memory**|易失、容量数百 GB 级|几十纳秒|临时数据处理，无法存完整数据库|
		|**Flash（闪存）**|非易失、读快写慢、有限擦除周期（10K-1M 次）|读≈主存，写 = 微秒级|嵌入式设备、SSD、U 盘|
		|**Magnetic Disk（硬盘）**|磁存储、直接访问、容量极大|毫秒级（寻道 + 旋转延迟）|数据库长期存储（完整数据库），单盘最大约 44TB|
		|**Optical Disk（光盘）**|激光读取、非易失|慢于硬盘|数据分发、离线存储（CD/DVD/ 蓝光）|
		|**Magnetic Tape（磁带）**|顺序访问、极低单位成本|最慢|数据备份、海量归档（Oracle SL8500 库可达 1.2EB）|
# 磁盘
- 物理结构：参见[[L4.1 存储器体系结构#^e6f3f2|计算机组成原理——非易失存储器的类型]]
- 访问时间：参见[[L4.1 存储器体系结构#^957f04|计算机组成原理——磁盘访问时间]]
	- *注1：磁盘内道的速率比外道低，因为内道的线速度较慢*
	- *注2：接口传输率也会影响访问时间，尤其是对于大数据量的读写操作*
- 磁盘可靠性：平均失效间隔（MTTF）
	- 磁盘连续无故障运行的平均时间
		- 新盘的理论MTTF在50万-120万小时，对应3-5年的使用寿命
	- 故障规律：在使用早期和硬件老化的后期故障率上升
- 磁盘调度优化：
	- 盘块与盘臂调度优化：
		- 盘块（Block）：
			- 单个磁道上连续的扇区组成的**数据传输的基本单位**
			- 大小：一般在4k-16kB之间
				- 块过小：增加I/O次数，降低效率
				- 块过大：产生内部碎片，浪费存储空间
		- 盘臂调度优化：减少磁头的移动，降低寻道时间
			- **电梯算法**：磁头在磁道之间来回移动，优先服务当前移动方向上的请求，直到没有更多请求，然后反向移动
	- 文件碎片与优化：
		- 文件碎片：文件随操作，存储块分布在磁盘不同位置，增加寻道时间
		- 优化：
			- 将关联数据存储在相近的物理位置
			- 定期进行磁盘碎片整理，重新排列文件以减少碎片
# 存储访问
- 为高效访问存储，需要缓存机制
	- 块（Block）：数据传输的基本单位，通常为4KB-16KB
	- 缓存（Buffer）：内存中的临时存储区域，用于存储从磁盘读取的数据块
	- 缓存管理器（Buffer Manager）：负责管理缓存，处理数据块的加载、替换和写回
- 缓存管理器的工作原理：
	- 程序访问磁盘块时，会先向缓存管理器发起请求：
		- 命中（Hit）：请求的数据块已在缓存中，直接返回数据，无需磁盘I/O
		- 缺失（Miss）：请求的数据块不在缓存中，需要从磁盘读取数据块到缓存中，然后返回数据
			1. 分配缓存空间：如果缓存有空余，直接分配；如果缓存已满，需要选择一个块进行替换
				- 被替换的块，只有被修改过时（脏位为1），才需要写回磁盘，称为**延迟写回**策略
			2. 读入数据块：从磁盘读取数据块到缓存中，更新缓存状态
			3. 返回地址：返回新加载的数据块在缓存中的地址给程序
    - 缓存替换策略：
	    - LRU（Least Recently Used）：优先替换最近最少使用的块
		    - 优点：适用于访问模式具有时间局部性的情况
		    - 局限：当嵌套循环访问时，可能会频繁替换块，导致性能下降
		- Toss-Immediate（立即销毁）：每次访问后立即销毁块
            - 适合嵌套循环中的外层关系块，避免频繁替换块
        - MRU（Most Recently Used）：优先替换最近最常使用的块
            - 适合嵌套循环中的内层关系块，被反复扫描的块此时
        - 现代缓存管理：
	        - 多种替换策略结合使用
	        - 基于预测的策略：利用访问统计信息，将高频块常驻于缓存中
    - 并发场景安全访问：PIN/UNPIN操作
	    - PIN：将块固定在缓存中，防止被替换
        - UNPIN：解除块的固定状态，使其可以被替换
