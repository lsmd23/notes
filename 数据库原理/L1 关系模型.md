# 关系数据模型
- 基本概念：
	- 域（Domain）：同类型的允许值的集合
		- 特殊值：NULL（空值），表示未知或不适用的值
		- 例：年龄域：{0, 1, 2, ..., 120}
	- 关系（Relation）：域的[[L7 关系#^80ff0d|笛卡尔积]]的一个子集。给定域$D_1, D_2, ..., D_n$，关系R是$D_1 \times D_2 \times ... \times D_n$的一个子集
		- 元组（Tuple）：关系中的一个元素，表示一个具体的记录$(a_1, a_2, ..., a_n)$，其中$a_i$是域$D_i$中的一个值\
		- 元数（Arity）：关系中的属性数量，即关系的列数
		- 例：
			- 域：学生ID域、姓名域、年龄域
			- 关系：学生表，包含元组(123, 'Alice', 20)，(456, 'Bob', 22)，元数为3
		- 性质：
			- 关系的每个元素，对应于关系型数据库表中的一行
			- 元组变量的访问：
				- 位置访问：`R[1]`表示关系R的第一个属性
				- 名称访问：`R['属性名']`表示关系R中属性名对应的值
			- 关系的无序性：关系中的元组没有特定的顺序
- 关系模式与实例：
	- 关系模式（Relation Schema）：关系的结构定义，包括关系名称和属性列表，表示为$R(A_1, A_2, ..., A_n)$，其中$R$是关系名称，$A_1, A_2, ..., A_n$是属性名称
		- 例：学生表的关系模式为Student(ID, Name, Age)
	- 关系实例（Relation Instance）：在某一时刻，关系的具体数据集合，即关系模式的一个具体实现，包含若干元组
		- 例：学生表的关系实例可能包含以下元组：(123, 'Alice', 20)，(456, 'Bob', 22)
	- 数据库模式（Database Schema）：数据库中所有关系模式的集合
		- 例：一个学校数据库可能包含学生表、课程表、成绩表等多个关系模式
	- 数据库实例（Database Instance）：在某一时刻，数据库中所有关系实例的集合
		- 例：一个学校数据库的实例可能包含学生表的具体数据、课程表的具体数据、成绩表的具体数据等
- 键：
	- 超键（Superkey）：设$K \subseteq R$，如果对于关系实例中的任意两个不同的元组$t_1$和$t_2$，满足$t_1[K] \neq t_2[K]$，则称$K$是关系模式R的一个超键。也即，能**唯一标识关系中的每个元组**的属性集合称为超键
		- 例：若学生表中，每个学生都不重名，则学生名字`student_name`或者学生ID `student_id`的组合`(student_name, student_id)`都是超键，因为它唯一标识每个学生
	- 候选键（Candidate Key）：在所有超键中，**最小的一个**，即没有冗余属性的超键
		- 例：学生表中，学生ID `student_id`是一个候选键，因为它唯一标识每个学生，并且没有冗余属性
	- 主键（Primary Key）：从候选键中**选择的一个**，用于唯一标识关系中的每个元组，通常在数据库设计中被指定为主键
		- 例：学生表中，学生ID `student_id`通常被指定为主键，因为它唯一标识每个学生
	- 外键（Foreign Key）：设关系$r_1(R_1)$有主键$K_1$，关系$r_2(R_2)$的属性子集$\alpha$若引用了$K_1$，且$r_2$中每个元组的$\alpha$值都能在$r_1$的$K_1$中找到对应值，则**称$\alpha$是$r_2$引用$r_1$的外键**。同时，称$r_2$为施引关系（referencing relation），$r_1$为被引关系（referenced relation）
		- 例：如果有一个课程表`Course(CourseID, CourseName)`和一个成绩表`Grade(StudentID, CourseID, Grade)`，其中`CourseID`是课程表的主键，`Grade`表中的`CourseID`是一个外键，引用了`Course`表的主键`CourseID`
# 关系代数基础
- 定义：由关系（集合）和关系运算符构成的[[Chapter 8 代数结构基础#代数系统的概念|代数系统]]
	- 关系代数可以用于数据库的查询语言，用于从数据库中查询数据
		- 过程化语言（Procedural Language）：用户需要指定查询的步骤和过程，如SQL，关系代数也是一种过程化语言
		- 非过程化语言（Non-Procedural Language）：用户只需要指定查询的目标和条件，如关系演算，关系演算是一种非过程化语言
- 关系代数的基本运算：
	- 并（Union）：$R \cup S = \{t | t \in R \vee t \in S\}$，即包含在R或S中的所有元组的集合
		- 限制：$r$和$s$必须元数相同，且对应属性的域必须相同
		- 例：![[Pasted image 20260304111757.png]]
	- 差（Difference）：$R - S = \{t | t \in R \wedge t \notin S\}$，即包含在R中但不包含在S中的所有元组的集合
		- 限制：同并运算
		- 例：![[Pasted image 20260304111819.png]]
	- 笛卡尔积（Cartesian Product）：$R \times S = \{t  q| t \in R \wedge q \in S\}$，即R中的每个元组与S中的每个元组组合成一个新的元组的集合
		- 限制：若存在重名属性，需要对属性进行换名操作以避免冲突
		- 例：![[Pasted image 20260304112020.png]]
	- 选择（Selection）：记$p$为选择[[L4 谓词逻辑的基本概念#^c0dfbb|谓词]]，则$\sigma_p(R) = \{t | t \in R \wedge p(t) \text{为真}\}$，即从关系R中选择满足条件p的所有元组的集合
		- 谓词$p$是一个布尔表达式，涉及关系R的属性
		- 例：![[Pasted image 20260304113458.png]]
	- 投影（Project）：$\Pi_{A_1, A_2, ..., A_k}(r) = \{t[A_1, A_2, ..., A_k] | t \in R\}$，即从关系R中选择指定属性$A_1, A_2, ..., A_k$的值，形成一个新的关系的集合
		- 例：![[Pasted image 20260304113543.png]]
	- 换名（Rename）：$\rho_X(A_1, A_2, ..., A_k)(E) = \{t[A_1, A_2, ..., A_k] | t \in R\}$，即将关系R的属性重命名为$A_1, A_2, ..., A_k$，形成一个新的关系的集合
		- 解决属性重名问题，特别是在笛卡尔积等操作中可能出现的属性冲突
- 关系代数表达式：
	- 递归定义：
		1) 关系R是一个关系代数表达式
		2) 如果E和F是关系代数表达式，则$E \cup F$, $E - F$, $E \times F$, $\sigma_p(E)$, $\Pi_{A_1, A_2, ..., A_k}(E)$, $\rho_X(A_1, A_2, ..., A_k)(E)$也是关系代数表达式
	- 例：复合运算$\sigma_{A=C}(r\times s)$表示从关系r和s的笛卡尔积中选择满足属性A等于属性C的元组的集合
	- 应用：
		- 银行数据库：![[Pasted image 20260304114946.png]]
		- 查询有贷款或存款的客户姓名：$$\Pi_{customer\_name}(\sigma_{Borrower.id=Customer.id}(Borrower \times Customer)) \cup \Pi_{customer\_name}(\sigma_{Depositor.id=Customer.id}(Depositor \times Customer))$$
- 关系代数的附加运算：
	- 交（Intersection）：$R \cap S = \{t | t \in R \wedge t \in S\}$，即包含在R和S中的所有元组的集合
		- 限制：同并运算
		- 例：![[Pasted image 20260304115126.png]]
	- 自然连接（Natural Join）：$R \bowtie S$，即在R和S的笛卡尔积的基础上，选择满足R和S中同名属性值相等的元组，并且只保留一个同名属性的值
		- 例：![[Pasted image 20260304115200.png]]$R = (A,B,C,D),S=(E,B,D)$，则$R \bowtie S = (A,B,C,D,E)$，其中$B$和$D$是R和S的公共属性，连接后只保留一个$B$和一个$D$，并且满足连接条件的元组才会出现在结果中
	- 条件连接（Condition Join，也称为Theta Join）：$R \bowtie_p S$，即在R和S的笛卡尔积的基础上，选择满足条件p的元组，并且保留所有属性
		- 例：![[Pasted image 20260304115501.png]]
